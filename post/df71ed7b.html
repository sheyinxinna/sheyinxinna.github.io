6.static关键字有什么作用

  1. 修饰内部类
    • 修饰的类是静态内部类
  2. 修饰方法
    • 静态方法
    • 不能被重写，可直接使用类名来调用
  3. 修饰变量
    • 静态变量
    • 被所有实例共享
  4. 修饰代码块
    • 静态代码块
    • 整个类加载的时候只会执行一次
    • 用来做程序优化

7.final的作用

  1. 修饰类
    • 修饰的类不能被继承
  2. 修饰方法
    • 修饰的方法不能被重写
    • JVM会尝试将其内联，提高运行效率
  3. 修饰变量
    • 修饰的变量不能被改变
    • 常量
    • 在编译阶段会被存入常量池中

8.StringString StringBuffffer 和 StringBuilder 的区别是什么

特性 String StringBuffer StringBulider
可变性 不可变 可变 可变
性能 较差 中等（受同步机制影响） 最佳
字符串拼接 创建多个中间String对象 修改通过一个StringBuffer对象 修改通过一个StringBulider对象
线程安全性 线程安全 线程安全 线程不安全
适用场景 不经常修改字符串内容 需频繁修改字符串内容 需频繁修改字符串内容

String：

  1. String是不可变的，一旦创建就不能修改其值。（只读字符串）
  2. 对于每次针对String的修改操作，实际上是创建了一个新的String对象。
  3. 字符串拼接使用”+”操作符时，会创建多个中间String对象，性能较低。
  4. String适用于在不经常修改字符串内容的场景，例如字符串常量、方法参数等。
  5. String类是线程安全的，可以保证多个线程同时读取一个字符串时不会出现问题。

StringBuffer：

  1. StringBuffer是可变的，可以修改其值。
  2. 对于每次针对StringBuffer的修改操作，不会创建新的StringBuffer对象，而是在原字符串缓冲区上进行修改。
  3. 字符串拼接时，不会创建多个中间的String对象，性能较高。
  4. StringBuffer适用于需要频繁修改字符串内容的场景，例如循环拼接字符串、多线程下的字符串操作等。
  5. StringBuffer类是线程安全的，提供了同步的方法，可以在多线程环境下使用。(Synchronized)

StrigBuffer是如何实现线程安全的

  1. StringBuffer通过内部使用同步机制来实现线程安全的
  2. 具体来说，就是它的每个方法上都使用了synchronized关键字，以确保在同一时间只有一个线程可以访问和修改StrigBuffer对象

StringBuilder：

  1. StringBuilder和StringBuffer一样也是可变的，可以修改其值。
  2. StringBuilder相比于StringBuffer，不进行同步，因此效率更高。
  3. 字符串拼接时，不会创建多个中间的String对象，性能最高。
  4. StringBuilder适用于在单线程环境下频繁修改字符串内容的场景。
  5. StringBuilder类不是线程安全的，不提供同步的方法。

10. 抽象类和接口的区别

特征 抽象类 接口
定义 一个包含抽象方法的类 一个行为规范的集合，描述了一个对象应该具有的方法
关键字 abstract interface
实例化 不能直接实例化，需要通过创建子类并实现所有抽象方法后才能创建对象 不能直接实例化，需要通过实现接口的类来创建对象
方法 可以包含普通方法和抽象方法 只能包含抽象方法和默认方法
继承 子类可以继承一个抽象类，并且可以扩展其属性和方法 一个类可以实现多个接口，并且可以在接口之间进行多重继承
实现限制 一个类只能继承一个抽象类 接口没有字段或实例变量，只能定义常量

总结：抽象类是一个类的抽象形式，可以包含普通方法和抽象方法，而接口是一个行为规范的集合，只能包含抽象方法和默认方法。一个类可以继承一个抽象类，但可以实现多个接口。抽象类的目的是为了提供代码的复用性和继承关系的建立，而接口的目的是为了实现多态和行为规范的一致性

13.在使用jdbc时，如何防止SQL注入

使用PrepareStatement,而不是使用Statement

PreparedStatement：

  1. 预编译：PreparedStatement使用预编译技术，将SQL查询语句在执行之前进行编译。这样可以提高查询的性能，特别是当需要多次执行相同的SQL语句时。
  2. 参数化查询：PreparedStatement支持参数化查询，可以在SQL语句中使用占位符（?）代表输入参数，然后使用set方法将具体的参数值设置到对应的占位符中。
  3. 防止SQL注入：由于参数化查询，PreparedStatement对于防止SQL注入攻击提供了更好的保护。参数值被视为纯数据，而不是SQL代码的一部分，因此不会被误解释为可执行的SQL语句。
  4. 可读性和维护性：由于使用了占位符，PreparedStatement通常比较易读且易于维护，可以在SQL语句和参数值之间进行更清晰的分离。

Statement：

  1. 即时编译：Statement在执行SQL查询之前会进行即时编译，每次执行都需要重新编译一次SQL语句。
  2. 字符串拼接：Statement通常使用字符串拼接的方式将参数值直接嵌入到SQL语句中，这样容易导致SQL注入攻击，因为恶意输入可能会被解释为SQL代码。
  3. 灵活性：由于没有占位符的支持，Statement在构建SQL查询时更加灵活。可以直接在SQL语句中嵌入参数值和SQL代码。
  4. 性能：由于没有预编译的过程，Statement在某些情况下可能会比PreparedStatement的执行速度稍慢。特别是在需要多次执行相同的SQL语句时，Statement可能会导致每次执行都进行语法解析和优化。

14.什么是连接池？

连接池是用来分配、管理、释放数据库连接的。

可以使应用程序重复使用同一个数据库连接，而不是每次都创建一个新的数据库连接。

通过释放空闲时间较长的数据库连接避免数据库因为创建太多的连接而造成的连接遗漏
问题，提高了程序性能。
